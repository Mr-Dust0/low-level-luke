# Agents Guide: Braindumps

Use this file as the agent-facing equivalent of `CLAUDE.md` for the braindumps project area.

## Purpose

Braindumps are active-recall exercises:

1. User writes raw notes from memory.
2. Keep those raw notes exactly as written in `## My Notes`.
3. Correct mistakes in `## What I Got Wrong`.
4. Write the full explanation in `## How [Topic] Actually Works`.

## File Rules

- Location: `/content/braindumps/`
- Name: `topic-name.md` (or `topic-name/index.md` if assets are needed)
- Date: set to the day the braindump is written
- Tags: include `"braindump"` first

## Required Structure

Every braindump must follow:

1. Frontmatter
2. Intro italic line
3. `## My Notes` (verbatim user text)
4. `## What I Got Wrong` (specific corrections)
5. `## How [Topic] Actually Works` (clean explanation)
6. `### Tradeoffs` (table when relevant)
7. Closing line: `P.S. If there are any mistakes please let me know, I'm by no means an expert.`

## Correction Style

- Quote/paraphrase what was wrong, then correct it.
- Explain why the distinction matters.
- Include missing concepts, not only wrong claims.
- Be direct, specific, and non-condescending.

## Repo Workflow Reminder

From project root (`/home/john/blog`), after content/config changes:

1. Run `hugo`.
2. Commit changes (including `public/` when applicable).
3. Push.
